# Glossário do MandaJá

| Termo | Definição |
| --- | --- |
| Usuário | Pessoa autenticada no MandaJá, associada a um ou mais perfis e escopos. |
| Cliente | Usuário que pesquisa produtos, cria pedidos e acompanha suas compras. |
| Motoboy | Usuário aprovado para coletar e entregar pedidos, próprio de um estabelecimento ou autônomo. |
| Estabelecimento | Empresa ou profissional autorizado a publicar produtos e atender pedidos. |
| Dono | Responsável que administra somente seus estabelecimentos e equipes. |
| Gerente | Usuário vinculado que administra operações delegadas pelo dono. |
| Funcionário | Usuário operacional que visualiza e atualiza etapas de preparo permitidas. |
| Caixa | Usuário autorizado a tratar etapas financeiras operacionais permitidas. |
| Estoque | Perfil responsável por produtos, quantidades, disponibilidade e alertas. |
| Dev N1 | Administrador máximo da plataforma, com intervenções sensíveis auditadas. |
| Pedido | Registro comercial imutável em seus valores-base, com itens, atendimento, pagamento e histórico de status. |
| Entrega | Processo logístico associado a um pedido, desde a oferta até a confirmação. |
| Assinatura | Contrato de plano do estabelecimento, com ciclo, valor e situação financeira. |
| Pagamento | Tentativa ou confirmação financeira de pedido ou assinatura. |
| Comissão | Parcela calculada para a plataforma conforme regra vigente registrada no momento da transação. |
| Repasse | Valor devido ao estabelecimento ou motoboy após descontos e condições aplicáveis. |
| Avaliação | Nota e comentário feitos após uma experiência elegível e vinculada a pedido concluído. |
| Promoção | Regra temporária de preço ou destaque aplicada a produtos elegíveis. |
| Cupom | Benefício identificado por código, sujeito a validade, limite e regras de uso. |
| Chamado de suporte | Solicitação de atendimento com categoria, prioridade, mensagens e histórico. |
| Log de auditoria | Registro protegido de quem fez o quê, quando, em qual recurso e com qual resultado. |
| Rastreamento | Compartilhamento temporário da posição do motoboy durante uma entrega ativa. |
| Código de entrega | Segredo de uso limitado que confirma a entrega ao destinatário autorizado. |
| Pedido agendado | Pedido preparado ou entregue para data e horário futuros validados. |
| Retirada local | Atendimento em que o cliente coleta o pedido no estabelecimento. |
| Região atendida | Área geográfica em que um estabelecimento ou operação aceita entregas. |
| Taxa de entrega inteligente | Valor calculado por regras como distância, área, horário e configuração vigente. |
| Antifraude | Processo de detectar, pontuar e tratar comportamentos suspeitos sem decisão discriminatória automática. |
| Comprovante | Documento informativo do pedido ou pagamento; não é documento fiscal por definição. |
| Documento fiscal | Documento sujeito à legislação tributária e emitido por solução fiscal habilitada. |
| Idempotência | Propriedade que permite repetir uma operação sem duplicar seu efeito. |
| Backup | Cópia ou exportação independente usada para recuperação após perda ou corrupção. |
| RPO | Perda máxima de dados, medida em tempo, aceita após um incidente. |
| RTO | Tempo máximo esperado para restaurar um serviço após um incidente. |
| Custom claim | Atributo controlado no Firebase Authentication usado como sinal de autorização. |
| App Check | Proteção que ajuda a bloquear clientes não autorizados no acesso aos recursos Firebase. |
| Dado pessoal | Informação relacionada a pessoa natural identificada ou identificável. |
| Dado pessoal sensível | Categoria especial da LGPD que exige proteção e hipótese legal adequadas. |
| Titular | Pessoa natural a quem os dados pessoais se referem. |
| Controlador | Agente que decide as finalidades e os meios do tratamento de dados pessoais. |
| Operador | Agente que trata dados pessoais em nome do controlador. |
